完成,常规的非懒加载的单例 bean 的实例化

finishBeanFactoryInitialization()

  • 给 bf 设置 ConversionService 和 EmbeddedValueResolver
  • 停用 tempClassLoader
  • 冻结 bd,做好实例化准备
protected void finishBeanFactoryInitialization(ConfigurableListableBeanFactory beanFactory) {
		//如果存在ConversionService的bean,给bf设置ConversionService,
    	用于类型转换.
		if (beanFactory.containsBean(CONVERSION_SERVICE_BEAN_NAME) &&
				beanFactory.isTypeMatch(CONVERSION_SERVICE_BEAN_NAME, ConversionService.class)) {
			beanFactory.setConversionService(
					beanFactory.getBean(CONVERSION_SERVICE_BEAN_NAME, ConversionService.class));
		}

		//如果bf还没有内置的EmbeddedValueResolver话,给bf设置一个,
    	类型是StringValueResolver,(如果你的ac没有BeanFactoryPostProcessor的话,就用他来帮你处理占位符)
		if (!beanFactory.hasEmbeddedValueResolver()) {
			beanFactory.addEmbeddedValueResolver(strVal -> getEnvironment().resolvePlaceholders(strVal));
		}

		// Initialize LoadTimeWeaverAware beans early to allow for registering their transformers early.
		//初始化LoadTimeWeaverAware beans,
    	此类bean用来在将类加载到Java虚拟机(JVM)中时动态地转换类,因此尽可能早的实例化,一般也用不着
		String[] weaverAwareNames = beanFactory.getBeanNamesForType(LoadTimeWeaverAware.class, false, false);
		for (String weaverAwareName : weaverAwareNames) {
			getBean(weaverAwareName);
		}

		//有些阶段会new ContextTypeMatchClassLoader(beanFactory.getBeanClassLoader())作为tempClassLoader
		beanFactory.setTempClassLoader(null);

		// Allow for caching all bean definition metadata, not expecting further changes.
		//冻结所有的bean定义，标志着已注册的bean定义将不会被修改或任何进一步的处理,
    	都要进行实例化了,你再修改bd哪还行?
		beanFactory.freezeConfiguration();

		// Instantiate all remaining (non-lazy-init) singletons.
		//真正开始初始化剩下的非懒加载的单实例，90%的bean都在此被实例化
		beanFactory.preInstantiateSingletons();
	}

preInstantiateSingletons()

  • 真正
DefaultListableBeanFactory.preInstantiateSingletons() {
//为bf的beanDefinitionNames创建一个副本集合
		List<String> beanNames = new ArrayList<>(this.beanDefinitionNames);

		//for循环beanName获取BeanDefinition
		for (String beanName : beanNames) {
			//返回RootBeanDefinition，如果是非RootBeanDefinition，则转换成RootBeanDefinition返回
			RootBeanDefinition bd = getMergedLocalBeanDefinition(beanName);
            //非抽象,单例,非懒加载才继续
			if (!bd.isAbstract() && bd.isSingleton() && !bd.isLazyInit()) {
				if (isFactoryBean(beanName)) {
                    //如果是工厂bean,则调用getBean()方法得到工厂bean的实例"&beanName"
					Object bean = getBean(FACTORY_BEAN_PREFIX + beanName);

					if (bean instanceof FactoryBean) {
						final FactoryBean<?> factory = (FactoryBean<?>) bean;
						boolean isEagerInit;
						if (System.getSecurityManager() != null && factory instanceof SmartFactoryBean) {
							isEagerInit = AccessController.doPrivileged((PrivilegedAction<Boolean>)
											((SmartFactoryBean<?>) factory)::isEagerInit,
									getAccessControlContext());
						} else {
							isEagerInit = (factory instanceof SmartFactoryBean &&
									((SmartFactoryBean<?>) factory).isEagerInit());
						}
						if (isEagerInit) {
                            //如果工厂bean是SmartFactoryBean类,而且是急切的,就直接触发getBean(beanName),得到目标bean
							getBean(beanName);
						}
					}
				} else {
                    //常规bean,直接触发
					getBean(beanName);
				}
			}
		}

		// 当所有单例bean都实例化完成之后,为所有实现SmartInitializingSingleton接口的bean,回调afterSingletonsInstantiated()
		for (String beanName : beanNames) {
			Object singletonInstance = getSingleton(beanName);
			if (singletonInstance instanceof SmartInitializingSingleton) {
				final SmartInitializingSingleton smartSingleton = (SmartInitializingSingleton) singletonInstance;
				if (System.getSecurityManager() != null) {
					AccessController.doPrivileged((PrivilegedAction<Object>) () -> {
						smartSingleton.afterSingletonsInstantiated();
						return null;
					}, getAccessControlContext());
				} else {
					smartSingleton.afterSingletonsInstantiated();
				}
			}
		}
	}

getMergedLocalBeanDefinition(String beanName)

通过 beanName 获取合并后的 bd,返回 RootBeanDefinition

AbstractBeanFactory#getMergedLocalBeanDefinition(String beanName){
//先从mergedBeanDefinitions缓存中拿,能拿到,而且mbd.stale是false(是否过期,陈旧),就直接返回
		RootBeanDefinition mbd = this.mergedBeanDefinitions.get(beanName);
		if (mbd != null && !mbd.stale) {
			return mbd;
		}
    //否则,上面没拿成功,就走下面逻辑去处理
		return getMergedBeanDefinition(beanName, getBeanDefinition(beanName));

}

//从mergedBeanDefinitions缓存中拿不到或者拿到了了但stale为true
AbstractBeanFactory#getMergedBeanDefinition(String beanName, BeanDefinition bd, @Nullable BeanDefinition containingBd){
    主要逻辑:
    1.加锁再从mergedBeanDefinitions拿一遍,还是拿不到,就往下继续
    2.根据你传入的bd,尝试获取它的parentBeanName(),
   		2.1如果有父pbd,则根据pbd,将pbd与你传入的原始bd合并,mbd.overrideFrom(bd);即以父pbd为模板,从bd中读取属性,覆盖到pbd中,从而得到mbd.
        2.2如果没有父pbd,则,直接从传入的原始bd中直接克隆一份/或直接new RootBeanDefinition(bd),即得到最终的mbd.
    3.mbd存入mergedBeanDefinitions缓存.

}

mergedBeanDefinitions:存的是合并后的 RootBeanDefinition

private final Map<String, RootBeanDefinition> mergedBeanDefinitions;

  • 如果 beanName 第一次走到 doGetBean()方法,而且单例池中没获取到 bean(即首次创建 bean),则会经过 markBeanAsCreated(beanName);此方法中会将该 beanName 对应的 bd.stale=true,设置为过期
  • 在 refresh()方法的 invokeBeanFactoryPostProcessors()过程中,在最后清理阶段,beanFactory.clearMetadataCache();,这里会遍历将还未实例化的 beanName 对应的 bd.stale=true,设置为过期.因为 bfpp 的方法回调会修改 bd,必须告知缓存(当然这里不是完全清空缓存,而是设置一个标志位)
